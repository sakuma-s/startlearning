# 標準入出力を扱える

## 1. 標準入出力とは

標準入力、標準出力、標準エラー出力について、プログラミング初心者にわかるように説明してください。
#### 標準入力（stdin）
デフォルトでは、キーボードからのデータ入力を指します。プログラムはここからデータを受け取ります。
#### 標準出力(stdout)
デフォルトではコンソール画面（ターミナル等）へのデータ出力を指します。プログラムが正常な結果をここに出力します。
#### 標準エラー（stderr）
デフォルトでは標準出力と同じコンソール画面へ、プログラムのエラーメッセージや警告を出力します。
#### まとめ
標準入出力（Standard I/O）とは、プログラムが外部とデータをやり取りするための、あらかじめOS（オペレーティングシステム）によって用意されている3つの通信チャンネルこと。
## 2. リダイレクト

リダイレクトを使用して、ルートディレクトリを ls コマンドで出力した結果を、~/root.txt という新規ファイルに保存してください。

~~先ず、`touch root.txt`でファイル作成した後、`ls > root.txt` で新しいファイルに書き出す。~~
<br>ルートディレクトリは、最上位のディレクトリ。ダイレクト演算子 > や >> を使う場合、指定したファイル名（この場合は ~/root.txt）が存在しなければ自動で新規作成されます。
#### パスの指定:

root.txt を保存する場所は「ホームディレクトリ」と指定されているため、root.txt ではなく ~/root.txt と記述するのが正確です。
修正後のコマンド `ls / > ~/root.txt` で実行。
## 3. エラー出力のリダイレクト

コマンドを実行してエラーになった際に、エラー結果を ~/error.txt という新規ファイルに保存してください。エラーを発生させるにはは例えば ls /hoge など、存在しないファイル・ディレクトリを ls コマンドで参照する方法があります。

#### ls /hoge 2> ~/error.txt<br>
|ファイル・ディスクリプター番号|出力先|
|---------------------------|-----|
| 0 | 標準入力 |
| 1	| 標準出力 |
| 2	| 標準エラー出力 |
| n	| 任意の入出力先 |

## 4. 出力とエラー出力のリダイレクト

ルートディレクトリと存在しないファイルを ls コマンドで一度で参照しようとして( ls / /hoge )、その両方の結果を一度に ~/result.txt という新規ファイルに保存してください。<br>
`ls / /hoge > ~/result.txt 2>&1`

## 5. /dev/null とは

/dev/null とは何で、どういう時に使われるものかをプログラミング初心者にわかるように説明してください。

#### ・説明
/dev/null(デブ　ヌル)は、LinuxやUNIX系OSにおける**ゴミ箱**のような、非常に特別なファイルです。
#### 1.何であるか？
* 実際にはデータが保存されるファイルではなく、**「データを受け取っても即座に破棄する、仮想的なデバイス」** としてOSに認識されています。
* 簡単に言えば、**「入力されたデータをすべて捨て去る穴」**です。
>「分かりそう」で「分からない」でも「分かった」気になれるIT用語辞典<br>
ブラックホールのようなイメージ。
#### 2.どういう時に使われるか？
* `出力先として指定`されることがもっとも多い。プログラムが生成するメッセージやエラーを画面に表示させたくないとき、リダイレクト（`>`や`2>`）の保存先として`/dev/null`を指定します。
* 例:`ls /etc > /dev/null`(`etc`の内容を画面に表示せず、破棄する)

## 6. /dev/null へのリダイレクト

ルートディレクトリを ls コマンドで参照した結果を、/dev/null にリダイレクトし、何も表示されないことを確認してください。

## 7. パイプライン

ルートディレクトリを ls コマンドで参照した結果のうち、"l" から始まるものだけを、パイプラインを使用して一回のコマンドで表示してください。
